# 前记

上一章介绍完基本概念，这章将开始搭建服务。

主要从注册中心、发布服务、消费服务这几个方面进行。

# Eureka Server 服务注册中心

为了方便搭建，这里直接使用 Idea 中的 Spring Initializr 构建。

流程：
1. 先创建 Maven 主工程。
2. 然后创建两个 Module，一个作为 Server，一个作为 Client。（都是勾选 Cloud Discovery 中的 Eureka Server）
3. 在导入完依赖后就可以开始了。

![项目目录](project.png)

这里先在 Server 中进行操作。

添加注解 `@EnableEurekaServer`，使得 Spring Boot 应用成为服务注册中心：

```java
@EnableEurekaServer
@SpringBootApplication
public class ServerApplication {

    public static void main(String[] args) {
        SpringApplication.run(ServerApplication.class, args);
    }
}
```

在 `application.properties` 配置文件中写入：

```
# eureka server端口
server.port=8761

# eureka server的主机名
eureka.instance.hostname=localhost
# 当前服务不需要到eureka server上注册（默认情况下server也是一个client）
eureka.client.register-with-eureka=false
eureka.client.fetch-registry=false
# 供客户端注册的地址
eureka.client.service-url.defaultZone=http://localhost:${server.port}/eureka/
```

启动工程，打开 http://localhost:8761/ 即可进入 eureka server。

![Eureka Server](eureka_server.png)

可以看到标红的地方显示没有发现任何服务。

# Eureka Client 服务提供

现在来创建一个可以提供计算服务的客户端，实现一个 RESTFulAPI /add 接口，提供加法服务。

添加注解 `@EnableDiscoveryClient` 声明该应用是一个eureka client。

```java
@EnableDiscoveryClient
@SpringBootApplication
public class ClientApplication {

    public static void main(String[] args) {
        SpringApplication.run(ClientApplication.class, args);
    }
}
```

在 `application.properties` 配置文件中写入：

```
# eureka client端口
server.port=9090
# 消费者将通过该名称调用所提供的服务
spring.application.name=compute-service

# 在server的上注册，这里可以在多个server注册
eureka.client.service-url.defaultZone=http://localhost:8761/eureka/
```

接口：

```java
@RestController
public class ComputeController {

    private final Logger logger = LoggerFactory.getLogger(ComputeController.class);

    @Resource
    private EurekaInstanceConfig eurekaInstanceConfig;
    @Value("${server.port}")
    private Integer serverPort;

    @GetMapping("/add")
    public String add(@RequestParam Integer a, @RequestParam Integer b) {
        Integer after = a + b;
        logger.info("/add, instanceId:{} , host:{} , result:{}",
                eurekaInstanceConfig.getInstanceId(), eurekaInstanceConfig.getHostName(false), after);
        return String.format("Result: %d, from port: %d", after, serverPort);
    }

}
```

启动客户端，可以在控制台看到：

```
2018-05-26 11:04:53.865  INFO 8661 --- [nfoReplicator-0] com.netflix.discovery.DiscoveryClient    : DiscoveryClient_COMPUTE-SERVICE/localhost:compute-service:9090 - registration status: 204
```

# 服务消费

上面已经完成了计算服务的注册，接下来是消费其提供出来的接口。

Spring Cloud 有两种服务调用方式：
- Ribbon + RestTemplate
- Feign

## Ribbon + RestTemplate

同样，添加一个服务消费的 Module，不仅要添加 Cloud Discovery 中的 Eureka Server，还要添加 Cloud Routing 中的 Ribbon。

添加注解 `@EnableDiscoveryClient` 使得消费者可以发现服务；添加 RestTemplate Bean：

```java
@EnableDiscoveryClient
@SpringBootApplication
public class ConsumerApplication {

    public static void main(String[] args) {
        SpringApplication.run(ConsumerApplication.class, args);
    }

    @Bean
    // 开启负载均衡
    @LoadBalanced
    RestTemplate restTemplate() {
        return new RestTemplate();
    }
}
```

在 `application.properties` 配置文件中写入：

```
# 消费者端口
server.port=9990
# 消费者名称
spring.application.name=consumer-compute

# 和客户端一样，这里可以在多个server注册
eureka.client.service-url.defaultZone=http://localhost:8761/eureka/
```

创建接口，去消费 `COMPUTE-SERVICE` 提供的 add 服务：

```java
@Service
public class ComputeService {
    @Resource
    private RestTemplate restTemplate;

    public String add(Integer a, Integer b) {
        return restTemplate.getForEntity("http://COMPUTE-SERVICE/add?a=" + a + "&b=" + b, String.class).getBody();
    }
}
```

控制器：

```java
@RestController
public class ConsumerController {

    @Resource
    private ComputeService computeService;

    @GetMapping("/add")
    public String add(@RequestParam Integer a, @RequestParam Integer b) {
        return computeService.add(a, b);
    }
}
```

然后启动消费者，可以看到都注册上了：

![消费者](eureka_consumer.png)

访问多几次消费者的网址 http://localhost:9990/add：

```bash
curl http://localhost:9990/add\?a\=1\&b\=66
Result: 67, from port: 9091

curl http://localhost:9990/add\?a\=1\&b\=45
Result: 46, from port: 9090

curl http://localhost:9990/add\?a\=1\&b\=100
Result: 101, from port: 9091

curl http://localhost:9990/add\?a\=1\&b\=81
Result: 82, from port: 9090
```

可以看到两个 compute-service 客户端被交替调用。这是 Ribbon 在客户端已经实现了对服务调用的均衡负载。

![架构图](architecture.png)


- 一个服务注册中心 eureka server 端口为8761
- client 跑了两个实例，端口分别为9090、9091，分别向服务注册中心注册
- consumer 端口为9990,向服务注册中心注册
- 当 consumer 通过 restTemplate 调用 client 的 add 接口时，因为用 ribbon 进行了负载均衡，会轮流的调用 client：9090、9091 两个端口的 add 接口

## Feign

- Feign 基于接口的注解请求
- Feign 整合了 Ribbon

添加新的 Module，添加依赖：

```java
<dependencies>
    <dependency>
        <groupId>org.springframework.cloud</groupId>
        <artifactId>spring-cloud-starter-netflix-eureka-server</artifactId>
    </dependency>
    <dependency>
        <groupId>org.springframework.cloud</groupId>
        <artifactId>spring-cloud-starter-feign</artifactId>
        <version>1.4.4.RELEASE</version>
    </dependency>
</dependencies>
```

因为整合了 Ribbon，所以主程序中只需要注解 `@EnableFeignClients` 即可：

```java
@EnableFeignClients
@EnableDiscoveryClient
@SpringBootApplication
public class ConsumerApplication {

    public static void main(String[] args) {
        SpringApplication.run(ConsumerApplication.class, args);
    }
}
```

配置文件 `application.properties` 和上面的差不多：

```
server.port=9991

spring.application.name=consumer-feign-compute

eureka.client.service-url.defaultZone=http://localhost:8761/eureka/
```

定义消费的接口：

```java
// 消费 compute-service
@FeignClient("compute-service")
public interface ComputeService {

    // 请求服务中的 add 接口
    @GetMapping("/add")
    String add(@RequestParam(value = "a") Integer a, @RequestParam(value = "b") Integer b);
}
```

控制器：

```java
@RestController
public class ConsumerController {

    @Resource
    private ComputeService computeService;

    @GetMapping("/add")
    public String add(@RequestParam Integer a, @RequestParam Integer b) {
        return computeService.add(a, b);
    }
}
```
