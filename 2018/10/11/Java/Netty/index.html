    注意，这里我们并没有像 DISCARD 例子那样释放收到的消息，因为 Netty 已经在你写回消息时为你自动释放了（和流的概念差不多）。

    调用 write 并没有立刻将消息写回，而是在内部缓存起来了，还需要调用 flush 才能写回。你也可以使用更简洁的 writeAndFlush(msg) 方法。
